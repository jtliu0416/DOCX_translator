# Doctrans V0.2 更新日志

**发布日期：** 2026-04-24

---

## 新增功能

### 1. 多文件批量上传

支持一次拖拽或选择多个 DOCX 文件同时上传，系统自动为每个文件创建独立翻译任务。后台通过队列排队处理，同时最多运行 2 个翻译任务，其余自动排队等待，避免 API 限流。

每个文件独立显示进度条和状态，互不影响。

### 2. 翻译失败一键重试

翻译任务失败后，无需重新上传文件。在历史记录页面或翻译页面直接点击「重试」按钮，系统自动重置任务状态并重新执行翻译流程。

### 3. WebSocket 实时进度推送

翻译进度由原来的 2 秒轮询改为 WebSocket 实时推送，进度更新更加流畅及时。网络环境不支持 WebSocket 时自动回退为轮询模式。

### 4. 下载文件名优化

翻译完成后下载的文档文件名由原来的 `translated.docx` 改为 `原文件名_双语.docx`，例如上传 `产品报告.docx`，下载时文件名为 `产品报告_双语.docx`，方便归档管理。

---

## 改进项

| 项目 | V0.1 | V0.2 |
|------|------|------|
| 文件上传 | 单文件 | 多文件批量上传 |
| 进度更新 | HTTP 轮询（2秒间隔） | WebSocket 实时推送 |
| 失败处理 | 只能删除，重新上传 | 一键重试 |
| 下载文件名 | translated.docx | 原文件名_双语.docx |
| 并发控制 | 最多 3 个任务（超出拒绝） | 队列排队，最多 2 个同时翻译 |

## 配置变更

`backend/.env` 无新增配置项。

`backend/app/config.py` 新增：

```python
MAX_CONCURRENT_TRANSLATIONS = 2   # 同时翻译的最大任务数
MAX_PARALLEL_TASKS = 50           # 单用户最大排队任务数（原 3 → 50）
```

## 涉及文件

### 后端新增
- `backend/app/api/ws.py` — WebSocket 连接管理器与端点
- `backend/app/services/queue.py` — 全局翻译并发信号量

### 后端修改
- `backend/app/api/tasks.py` — 重试端点、下载文件名、排队信号量
- `backend/app/services/translator.py` — 进度更新时 WebSocket 广播
- `backend/app/main.py` — 注册 WebSocket 路由
- `backend/app/config.py` — 新增并发配置项

### 前端修改
- `frontend/src/views/Translate.vue` — 多文件上传、多任务进度、WebSocket
- `frontend/src/views/History.vue` — 下载文件名、重试按钮
- `frontend/src/api/index.js` — 新增 retryTask 接口
- `frontend/vite.config.js` — WebSocket 代理配置
